## [SoftwareCopyright-Skill](https://github.com/Fokkyp/SoftwareCopyright-Skill) #1

> 一款基于 Python 的软件版权与许可证合规性分析工具，能够自动扫描项目依赖并生成冲突报告。当前获得 596 个 Stars，适合开源项目维护者快速规避法律风险。

随着开源生态的快速发展，项目中大量引入第三方库导致许可证兼容性问题频发。**SoftwareCopyright-Skill** 的创建正是为了解决开发者「不知道自己用了哪些许可证、是否冲突」的痛点。项目采用静态代码分析与依赖树遍历相结合的方式，支持 `MIT`、`GPL`、`Apache 2.0` 等主流许可证的兼容性判断。

核心功能包括递归扫描本地目录、自动识别 `package.json`、`requirements.txt` 等配置文件，并输出可视化的许可证图谱。工具内置了 **SPDX 许可证列表**，能够标记互斥的许可证组合（如 `GPL-3.0` 与 `Apache-2.0` 的再分发冲突）。据开发者介绍，检测准确率在标准场景下超过 90%。

社区方面，该项目在 GitHub 上已有 596 个 Stars 和 136 个 Forks，活跃度处于中上水平。近期版本新增了 **CI/CD 集成模式**，支持在 GitHub Actions 中自动运行，并生成 Markdown 格式的合规报告。与同类工具 `licensee` 相比，SoftwareCopyright-Skill 更侧重冲突检测而非仅识别许可证类型。

适用场景包括：企业内审开源项目合规性、个人开发者发布前检查、以及开源基金会评估代码库。项目仅依赖 Python 3.8+ 和 `pip`，部署成本极低。未来计划加入许可证变更追踪功能，进一步降低法律风险。
